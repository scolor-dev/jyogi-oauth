スコープとデータ

クライアントが /oauth/authorize で要求できるスコープと、各スコープで取得可能なデータの対応です。

スコープ一覧

openid default

OpenID Connect 認証。トークンレスポンスに id_token が追加される。

エンドポイント返却されるクレーム
/oauth/userinfosub
id_tokeniss, sub, aud, exp, iat, auth_time, at_hash, nonce（指定時）

identity default

メンバーの基本識別情報（表示名、アイコン、テーマカラー、タグライン）。

エンドポイント返却されるクレーム
/oauth/userinfoname, picture, color, tagline
id_token同上
/api/v1/members/me/identitydisplay_name, avatar_url, theme_color, tagline

profile

メンバーの詳細プロフィール情報。

エンドポイント返却されるクレーム
/oauth/userinfoname, preferred_username, updated_at
id_tokenname, preferred_username

read default

リソースサーバーの読み取り権限。

エンドポイント許可される操作
/api/v1/*GET リクエスト

write

リソースサーバーの書き込み権限。

エンドポイント許可される操作
/api/v1/*POST, PUT, DELETE リクエスト
/api/v1/members/me/identityPUT（プロフィール更新）

identity:write

Identity の更新権限。write の代わりに Identity 更新のみ許可したい場合に使用。

エンドポイント許可される操作
/api/v1/members/me/identityPUT（プロフィール更新）

admin

管理者権限。管理 API へのアクセスに使用。通常のクライアントアプリでは要求しません。

スコープの組み合わせ例

ユースケースscope
ログインだけ（誰か確認）openid
ログイン + アイコン・名前表示openid identity
プロフィール閲覧アプリopenid identity read
プロフィール編集アプリopenid identity identity:write
サーバー間 API 連携read（client_credentials grant）

スコープの default について

default マークのスコープは、クライアント登録時にデフォルトで付与されます。クライアントが要求できるスコープは、管理者が設定した allowed_scopes の範囲内です。
